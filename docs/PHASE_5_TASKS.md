# Phase 3: CSS リファクタ + デザイン刷新(サイバーパンク基調)

## 目的

現状の CSS を構造的にリファクタしつつ、トーキョーN◎VA の世界観に合わせてシート・HUD・UI
全体のビジュアルデザインをサイバーパンク基調に刷新する。技術的な書き換え(フェーズ2
ApplicationV2 移行)が完了した、機能的に安定したシートに対して、構造と見た目の両方を
整える。

## 背景

フェーズ0〜B の間、シートは「機能優先」のデザインで実装されてきた。また CSS は段階的に
追加・修正が加えられてきたため、重複するレイアウト定義や整合しないスタイルが散在している。

本フェーズはデザイン刷新だけでなく **CSS の構造的リファクタ(重複定義の統合、整理)**も
含む。フェーズ2 で全シートが ApplicationV2 に書き直された後、まず CSS の骨格を整えてから
デザインを一気に適用することで、スパゲッティ化した CSS を二度蒸し返さない構造にする。

**HUD デザインの分担**: HUD の V2 化(構造)はフェーズ1、HUD のデザイン(見た目)は本フェーズで
担当する。構造と見た目を分離し、シートを二度蒸し返さない一気通貫を狙う。

デザイン刷新は技術的にはフェーズ2(ApplicationV2 移行)と同時にも可能だが、
スコープが膨らみすぎる・判断軸が増える等のリスクがあるため、独立したフェーズとして
分離する。

## 着手の前提条件

- 新フェーズ2(既存シートの ApplicationV2 移行)完了
- 全シートが ApplicationV2 で書き直されている
- 機能が安定している(EXP 計算・カード判定・神業基本動作が動作している)

## 著作権の申し送り

デザイン刷新時に以下を遵守すること(CLAUDE.md §1.3 / §4.3 の著作権方針に整合):

- 原作「トーキョーN◎VA THE AXLERATION」の**公式アート・固有のビジュアル意匠の模倣は避ける**。
- ジャンルとして一般的な**サイバーパンク表現**(ネオン・ホログラム・ダークテーマ等)に留める。
- フォント・アイコン・画像リソースは使用許諾を確認したもののみ使用する。

## サブフェーズ(暫定)

詳細はフェーズ2 完了時に確定する。現時点では大枠のみ記載。

### 3-0: デザイン方針の確定

実装ファイルは作成しない。以下の方針を議論・確定する。

- 全体のトーン(古典サイバーパンク / 現代風 / ミニマル等)
- カラーパレット(基調色・アクセント色・状態表現色)
- タイポグラフィ(見出し・本文・特殊用途のフォント選定)
- 装飾要素(ボーダー・グリッチ・スキャンライン・アイコン等の使用方針)
- アニメーション方針(あり / なし / 控えめに使う等)
- CSS リファクタ方針(ファイル分割 / CSS 変数体系の設計)

決定事項は `docs/DESIGN_REVIEW.md` に 3-0 として記録する。

### 3-1: CSS リファクタ + 共通スタイル基盤の整備

- 既存 CSS の重複定義・競合定義の洗い出しと統合
- CSS 変数の整理(カラー・スペーシング・タイポ等のデザイントークン化)
- 共通コンポーネント(ボタン・入力・カード等)のスタイル定義
- ベースとなる CSS ファイル構造の整理・リファクタ

### 3-2: シートごとのデザイン適用

- 各シートに対して、確定した方針を適用していく
- 順序はフェーズ2 で書き直したシートの優先度に従う

### 3-3: HUD / ダイアログ / その他 UI のデザイン更新

- TnxHud のデザイン更新(構造は新フェーズ1 で完了済み、ここでは見た目のみ)
- 各種ダイアログ(UsageCreationDialog 等)のデザイン更新

### 3-4: 検証と微調整

- 実機で動作確認しつつ、視認性・操作性の最終調整
- 必要に応じてアクセシビリティの確認(コントラスト等)

## 制約事項

- 機能的な書き換えは行わない(デザインの変更および CSS リファクタに留める)
- ApplicationV2 構造は壊さない
- DataModel への影響は与えない
- 著作権方針(本ドキュメント「著作権の申し送り」参照)を遵守する

## 担当時期

新フェーズ2(既存シートの ApplicationV2 移行)完了後に着手。
デザインイメージの詰めは、それまでの間に少しずつ進めてよい。
