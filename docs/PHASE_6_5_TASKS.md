# Phase 6.5: サイバーパンク基調のデザイン刷新

## 目的

トーキョーN◎VA の世界観に合わせて、シート・HUD・UI 全体のビジュアルデザインを
サイバーパンク基調に刷新する。技術的な書き換え(フェーズ6 ApplicationV2 移行)が
完了した、機能的に安定したシートに対して、見た目だけを当てていくフェーズ。

## 背景

フェーズ0〜6 の間、シートは「機能優先」のデザインで実装されてきた。プロジェクトの
最終形として、トーキョーN◎VA のサイバーパンクな世界観に整合した見た目に整える。

デザイン刷新は技術的にはフェーズ6(ApplicationV2 移行)と同時にも可能だが、
スコープが膨らみすぎる・判断軸が増える等のリスクがあるため、独立したフェーズとして
分離する。

## 着手の前提条件

- フェーズ6 完了(全シートが ApplicationV2 で書き直されている)
- 機能が安定している(神業・判定・戦闘進行が動作している)

## サブフェーズ(暫定)

詳細はフェーズ6 完了時に確定する。現時点では大枠のみ記載。

### 6.5-0: デザイン方針の確定

実装ファイルは作成しない。以下の方針を議論・確定する。

- 全体のトーン(古典サイバーパンク / 現代風 / 公式準拠 / ミニマル等)
- カラーパレット(基調色・アクセント色・状態表現色)
- タイポグラフィ(見出し・本文・特殊用途のフォント選定)
- 装飾要素(ボーダー・グリッチ・スキャンライン・アイコン等の使用方針)
- アニメーション方針(あり / なし / 控えめに使う等)

決定事項は `docs/DESIGN_REVIEW.md` に 6.5-0 として記録する。

### 6.5-1: 共通スタイル基盤の整備

- CSS 変数の整理(カラー・スペーシング・タイポ等のデザイントークン化)
- 共通コンポーネント(ボタン・入力・カード等)のスタイル定義
- ベースとなる CSS ファイル構造の整理

### 6.5-2: シートごとのデザイン適用

- 各シートに対して、確定した方針を適用していく
- 順序はフェーズ6 で書き直したシートの優先度に従う

### 6.5-3: HUD / ダイアログ / その他 UI

- TnxHud のデザイン更新
- 各種ダイアログ(UsageCreationDialog 等)のデザイン更新

### 6.5-4: 検証と微調整

- 実機で動作確認しつつ、視認性・操作性の最終調整
- 必要に応じてアクセシビリティの確認(コントラスト等)

## 制約事項

- 機能的な書き換えは行わない(デザインの変更に留める)
- ApplicationV2 構造は壊さない
- DataModel への影響は与えない

## 担当時期

フェーズ6(ApplicationV2 完全移行)完了後に着手。デザインイメージの詰めは、
それまでの間に少しずつ進めてよい。
