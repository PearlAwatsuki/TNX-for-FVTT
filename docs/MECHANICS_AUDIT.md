# MECHANICS_AUDIT

トーキョーN◎VA THE AXLERATION のメカニクス実装について、ルール解釈の正しさを監査する記録。

各エントリは「ルール解釈 → 実装 → 監査結果」の3要素で構成する。設計の技術的妥当性は
`DESIGN_REVIEW.md` 側で扱う。

## 運用ルール

- 新規実装時: その実装の範囲で1エントリ追加
- 既存実装に影響する変更時: 影響を受けるエントリのステータスを「再監査待ち」に戻す
- フェーズ完了時: そのフェーズで増えた全エントリを通読してステータスを最新化
- 1エントリの粒度: Claude Code に1つのプロンプトで投げる単位と一致させる
- ルール本文の引用や、特技名・神業名等の網羅的リスト化は行わないこと(著作権配慮)

## エントリテンプレート

```
### [機能名](フェーズX)

**日付**: YYYY-MM-DD
**関連実装**: `path/to/file.mjs` の `関数名` など
**ステータス**: 監査中 | 監査済(問題なし) | 監査済(課題あり) | 再監査待ち

#### ルール解釈
(ユーザーから聞き取った内容を、自分の言葉でメモ)

#### 実装
(どのファイルのどの処理として実装したか。コード本体は貼らず、要点だけ)

#### 監査結果
- ✅ 項目1 → 仕様通り
- ⚠️ 項目2 → 課題の説明
- ❌ 項目3 → ルール解釈と齟齬

#### 課題・申し送り
(再監査が必要な点、将来のフェーズで影響しそうな点、神業との相互作用など)
```
